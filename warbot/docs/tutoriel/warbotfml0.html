Présentation

Warbot est basé sur un formalisme, c'est à dire sur la description d'un modèle de structure. Nous renvoyons à la documentation interne à Madkit pour les détails de développement de formalismes (c'est très bien expliqué, en Anglais ;-), et nous nous contenterons ici d'en rappeler les grandes lignes.

Définition

Un formalisme est constitué de noeuds et de liens spécifiques d'un domaine. Ainsi, dans l'image ci-dessus, il s'agit de réseaux de Petri. La description de ces noeuds et liens est faite dans un arbre XML formalism:

  • formalism-info
  • node-types
    • node-desc*
    name
      • class
      • icon
      • ...
  • arrow-types
    • arrow-desc*
      • name
      • class
      • from
      • to
      • icon
      • ...
  • action
    • java-method

Les formalismes du projet WarBot

C'est sur des formalismes de ce type qu'est basée la définition de l'environnement WarBot. Une particularité importantee des formalismes de Warbot est qu'ils ne contiennent aucun lien, seulement des noeuds.
